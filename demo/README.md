2. 요구사항 정의서 작성 (The First Step)
가장 먼저 docs/requirements.md 파일을 만들고 아래 내용을 채워보세요. 이것이 본인의 개발 이정표가 됩니다.

[쇼핑몰 프로젝트 요구사항]
회원 관리

사용자는 회원가입 및 로그인을 할 수 있다. (Spring Security)

사용자는 일반 사용자(USER)와 관리자(ADMIN)로 나뉜다.

상품 관리

관리자는 상품을 등록, 수정, 삭제할 수 있다.

사용자는 상품 목록을 조회하고 상세 내용을 볼 수 있다.

주문 및 장바구니

사용자는 상품을 장바구니에 담을 수 있다.

사용자는 장바구니의 상품을 주문할 수 있으며, 주문 시 재고가 감소한다.

알림 (특화 기능)

주문 완료 시 실시간 브라우저 알림이 발송된다.

재고가 소진되면 관리자에게 알림이 간다.

3. GitHub Issue & Milestone 활용하기 (강력 추천)
현업에서는 Jira나 GitHub Issues를 사용해 할 일을 관리합니다. 이걸 포트폴리오에 녹여내면 **"협업 준비가 된 개발자"**라는 인상을 줍니다.

Milestones: 1단계: 회원 시스템, 2단계: 상품/주문, 3단계: 실시간 알림으로 나눕니다.

Issues: "회원 가입 API 구현", "DB 테이블 설계" 등 작은 단위의 할 일을 생성하고 Milestone에 연결합니다.

Commits: 커밋 메시지에 #Issue번호를 적으면 자동으로 연결됩니다. (예: feat: 회원가입 기능 구현 #1)

4. 데이터베이스 설계 (ERD 시각화)
코드를 짜기 전, 데이터들이 어떻게 연결될지 그림으로 그려야 합니다.

1:N 관계: 한 회원(Member)은 여러 번의 주문(Order)을 할 수 있습니다.

N:M 관계: 하나의 주문에는 여러 상품이 담기고, 한 상품은 여러 주문에 포함될 수 있습니다. (이 사이에 OrderItem이라는 중간 테이블을 두는 것이 JPA의 정석입니다.)
